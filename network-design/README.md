# 校园中小型网络工程设计与实现

## 1. 项目概述

本项目为某大学智慧校园网络设计，覆盖教学区、办公区、图书馆、学生宿舍、服务器区和管理区六大区域，采用经典的三层网络架构（核心层、汇聚层、接入层），实现VLAN隔离、子网划分、DHCP分配、ACL访问控制和静态NAT映射等功能。

## 2. 设计目标

- 不同区域之间通过VLAN实现二层隔离
- 核心交换机通过SVI接口实现VLAN间三层路由
- 各区域终端可通过DHCP自动获取IP地址
- 服务器区部署DNS、WWW、FTP、MAIL服务
- 出口路由器配置ACL实现访问控制
- 内网终端可通过NAT访问外网

## 5. 拓扑结构

采用三层网络架构：

- **核心层**：1台三层交换机（Cisco 3560），负责VLAN间路由和高速转发
- **汇聚层**：2台二层交换机，连接各区域接入交换机
- **接入层**：多台二层交换机，连接终端PC
- **出口路由器**：连接外网（ISP），配置NAT和ACL

## 6. 关键配置策略

### VLAN间路由
核心交换机上创建各VLAN的SVI接口（interface vlan），启用`ip routing`，实现跨VLAN通信。

### DHCP
核心交换机上配置DHCP池，为各VLAN自动分配IP地址，排除网关IP和服务器IP。

### ACL
出口路由器上配置扩展ACL：
- 允许内网访问外网（permit ip any any）
- 拒绝外网主动访问内网（在接口上应用ACL）
- 仅允许外网访问服务器的特定端口（WWW:80, FTP:21）

### NAT
出口路由器上配置PAT（端口地址转换），使得内网所有终端共享一个公网IP访问Internet。

## 7. 调试测试要点

1. 同VLAN内PC之间可以ping通
2. 不同VLAN之间可以ping通（经核心交换机路由）
3. 所有PC可以ping通服务器区各服务器
4. 内网PC可以通过NAT访问外网
5. ACL规则生效，外网无法直接访问内网
6. DHCP客户端成功获取IP地址
