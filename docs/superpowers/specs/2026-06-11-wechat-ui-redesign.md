# 局域网聊天软件 — 微信风格 UI + 聊天记录持久化

**日期**: 2026-06-11  
**状态**: 已确认

---

## 1. 目标

将现有 tkinter 聊天客户端重构为微信桌面版风格的界面，并在服务端增加聊天记录持久化，新登录用户可查看历史消息。

## 2. 整体架构

```
client.py (tkinter, 微信风格 UI)  ←→  server.py (多线程)
      ↓                                      ↓
  会话列表 + 聊天气泡                  messages.json (新增, 消息持久化)
                                      users.json (现有, 用户数据)
```

C/S 架构不变，TCP/JSON 协议扩展 2 个消息类型。

│  ┌───────────┐ │  ┌──────────────────────────┐   │

- **左侧面板** (180px): 搜索栏 + 会话列表。公聊大厅固定置顶（星标），下方按最近聊天时间排列私聊联系人
- **右侧面板**: 标题栏（当前会话名）+ 聊天气泡滚动区 + 底部输入栏
- **聊天气泡**: Canvas 绘制圆角矩形 + 多行文本。己方绿色 `#95EC69` 靠右，对方白色靠左
- **配色**: 微信风格，主色调 `#2DC100`，背景 `#F5F5F5`，左侧面板 `#EBEBEB`

## 4. 新增协议

```python
TYPE_GET_HISTORY = "get_history"  # 客户端 → 服务端, {"target": "public" | "alice"}
TYPE_HISTORY     = "history"      # 服务端 → 客户端, {"target": "...", "messages": [...]}
```

所有消息新增 `timestamp` 字段（ISO 格式字符串），登录成功响应中附带公聊最近 50 条历史。

## 5. 服务端变更

### 5.1 messages.json 结构

```json
{
  "public": [
    {"sender": "alice", "content": "大家好", "timestamp": "2026-06-11T10:00:00"}
  ],
  "alice:bob": [
    {"sender": "alice", "content": "私聊内容", "timestamp": "2026-06-11T10:01:00"}
  ]
}
```

- 公聊 key = `"public"`
- 私聊 key = 两个用户名按字母序用 `:` 连接（确保唯一）
- 每条消息存储 sender、content、timestamp

### 5.2 文件改动

| 位置 | 改动 |
|------|------|
| `__init__` | 新增 `_load_messages()` 加载 messages.json |
| `_handle_broadcast` | 发送后调用 `_save_message("public", ...)` 写入磁盘 |
| `_handle_private` | 发送后调用 `_save_message("alice:bob", ...)` 写入磁盘 |
| 新增 `_handle_get_history` | 根据 target 返回对应会话最近 N 条消息 |
| `_handle_login` | 响应中附带公聊最近 50 条历史 |
| `_dispatch` | 新增 `get_history` 分支 |

### 5.3 消息数量限制

每个会话最多保留 500 条消息，超出后淘汰旧消息，防止 JSON 文件无限增长。

## 6. 客户端变更

### 6.1 文件级改动

| 组件 | 说明 |
|------|------|
| `LoginWindow` | 保持不变（不修改） |
| `ChatWindow` | 完全重写，双面板布局，会话列表 + 聊天气泡 |
| 新增 `_build_left_panel` | 搜索框 + 会话列表 (Listbox) |
| 新增 `_build_right_panel` | 标题栏 + 气泡区 (Canvas + Scrollbar) + 输入栏 |
| 新增 `_render_bubble` | Canvas 绘制圆角矩形 + 自动换行文本 |
| 新增 `_load_history` | 请求并渲染聊天记录 |
| 修改 `receive_loop` | 处理 history/history 消息，解析 timestamp |
| 修改 `send_message` | 附带 timestamp，本地气泡立即渲染 |

### 6.2 登录流程

1. 登录成功 → 服务端返回 `{status:"ok", public_history: [...], conversations: ["bob", "charlie"]}`
   - `public_history`: 公聊最近 50 条
   - `conversations`: 该用户参与过的私聊对象列表（从 messages.json 的 key 中提取）
2. 客户端渲染左侧面板：「公聊大厅」(置顶) + 私聊联系人列表（按最近消息时间排序）
3. 默认选中「公聊大厅」，显示公聊历史
4. 收到在线用户列表后刷新在线状态标记

### 6.3 切换会话

- 点击「公聊大厅」→ 清空气泡区 → 请求公聊历史 → 渲染
- 点击私聊联系人 → 清空气泡区 → 请求该会话历史 → 渲染
- 新私聊（双击在线用户 / @用户名）→ 自动创建会话入口 → 加入左侧列表

### 6.4 气泡渲染规则

- 自己的消息：绿色 `#95EC69`，右对齐
- 别人的消息：白色 `#FFFFFF`，左对齐
- 系统消息：灰色小字居中
- 每条气泡下方显示时间戳（HH:MM 格式）
- Canvas 自动计算文本宽度换行，气泡高度随内容自适应

## 7. 不做的

- 群聊功能（保留公聊大厅作为替代）
- 文件/图片发送
- 表情包
- 消息加密
- 用户头像

## 8. 测试要点

1. 服务端重启后消息不丢失
2. 新登录用户能看到之前的公聊记录
3. 私聊记录按会话隔离，不会串
4. 快速连续发送多条消息，气泡正常渲染（不丢消息）
5. 切换会话时历史正确加载
6. 消息超过 500 条后旧消息被淘汰
7. LoginWindow 功能不受影响
